Quickstart

Working agent in under a minute

Default flow: clone the repo, install in editable mode, start Ollama, then run the bundled basic example or the CLI. 
git clone https://github.com/GameChaser782/agent-starter.git
cd agent-starter
pip install -e .
copy .env.example .env
ollama serve
ollama pull qwen3.5
python examples/01_basic/run.py

What you get

Agent orchestration, persistent memory, built-in tools, CLI commands, and a FastAPI surface.

Main entry point

AgentKit is the package entry point exported from agent_starter.

Version

Current package version in this repository: 0.1.0.

Installation

Choose local-first by default, add providers as needed

Core Usage

Start simple, then extend with your own tools and agents

Minimal Python usage

from agent_starter import AgentKit

agent = AgentKit()
thread_id = agent.new_thread()

response = agent.chat("Summarize why LangGraph is useful", thread_id=thread_id)
print(response)

Register a custom tool

from agent_starter import AgentKit

agent = AgentKit()

@agent.tool
def get_weather(city: str) -> str:
    """Get the current weather for a city."""
    return f"It's 22C and sunny in {city}."

print(agent.chat("What is the weather in Tokyo?"))

CLI commands

agent-starter chat --agent default
agent-starter chat --agent examples/research_assistant
agent-starter serve --agent default --port 8000

Built-in Tools

Enable only the tools your agent actually needs

calculator

Safe math expression evaluation.

file_reader

Read local text files.

bash_exec

Run shell commands and capture output.

code_editor

Read, write, and replace text in files.

python_repl

Execute Python code in a subprocess.

sqlite_query

Run SQL against local SQLite databases.

web_search

Search the web, optionally powered by Tavily.

github_read_file

Read files from GitHub repositories.

github_list_issues

List repository issues.

github_get_issue

Fetch issue or pull request details.

image_vision

Analyze local or remote images with a vision-capable model.

Enable tools in YAML

tools:
  - calculator
  - file_reader
  - web_search
  - github_read_file

API Server

Serve the same agent behind FastAPI

agent-starter serve
# http://localhost:8000
# http://localhost:8000/docs

POST /chat

Returns a full response string for a message, thread, and user.

POST /stream

Streams tokens over server-sent events and ends with [DONE].

GET /memory/{user_id}

Returns stored long-term memories when that feature is enabled.

GET /health

Simple health response for probes and local checks.

Sample request

curl -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"message":"Hello","thread_id":"t1","user_id":"alice"}'

Configuration

Agent behavior lives in YAML plus persona markdown

name: default
persona: default_persona.md

model:
  provider: ollama
  name: qwen3.5:4b
  temperature: 0.7
  thinking: false

tools:
  - calculator
  - file_reader

memory:
  long_term: true
  dir: .agentkit
  thread_ttl: 3600

server:
  host: "0.0.0.0"
  port: 8000

Env overrides

AGENT_PROVIDER, AGENT_MODEL, AGENT_TEMPERATURE, AGENT_LONG_TERM_MEMORY, and server vars can override YAML.

Persona source

The persona file is loaded from the agent config directory and injected into the graph as plain markdown text.

Memory directory

Long-term memory defaults to .agentkit and uses SQLite storage.
